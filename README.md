# LBM_post_process V1.0.0

这是一个用于验证多相标签体表面重建和平滑效果的桌面程序。它读取多页 TIFF，并使用接近 ParaView 的三栏布局：

- 左侧：相列表、显示开关、颜色、透明度、平滑和光照开关（这里不再显示第二个模型）
- 中央：当前裁剪工作区域的多相平滑三维视图和 3D 文字标注
- 右侧：持久化设置预设、数据、边框、裁剪、当前相属性、3D 标注和逐层计算模块

程序不会对相编号做线性插值。所有相都作为离散标签交给 VTK Surface Nets 重建。

## 安装与运行

Windows PowerShell：

~~~powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python phase_viewer.py --demo
~~~

Linux 或 macOS：

~~~bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
python phase_viewer.py --demo
~~~

在 PyCharm 中可以直接右键运行 phase_viewer.py。不填写参数时会自动打开内置三相演示体。

## 打开自己的 TIFF

从界面点击“打开 TIFF”，或者使用命令行：

~~~bash
python phase_viewer.py your_labels.tif
~~~

如果真实体素尺寸为 X=0.4、Y=0.4、Z=1.2：

~~~bash
python phase_viewer.py your_labels.tif --spacing 0.4 0.4 1.2
~~~

也可以在右侧 Properties 面板修改 X/Y/Z 体素尺寸，然后点击“重建所有可见相”。

## 防误触与设置预设

右侧 Properties 面板默认启用“右侧滚动防误触”。鼠标滚轮或触控板位于数值框、下拉框上时，只会继续滚动整个 Properties 面板，不会改变当前参数。参数仍可通过单击按钮、键盘输入、方向键或明确拖动来修改；如确实需要使用滚轮改值，可以取消该开关。

“设置预设”支持输入中文或英文名称，并执行“保存 / 覆盖”“加载”和“删除”。预设保存在当前操作系统的用户应用配置目录中，关闭程序或重新启动电脑后仍然存在。加载预设前先打开任意一个目标 TIFF，然后从列表选择名称并点击“加载”。

每个预设包含：

- 全部相编号的显示状态、颜色、透明度、平滑和光照参数
- X/Y/Z 体素尺寸、预览降采样、背景、边框和三角网格边
- 当前裁剪范围
- 全部 3D 标注及其坐标、颜色、字号、字体、加粗和斜体
- 中央三维模型的完整相机视点、焦点、朝上方向和投影参数
- 截图倍率及 Slice Volume Fraction 的方向、分子和分母设置
- 右侧滚动防误触开关

预设可直接用于原始矩阵尺寸相同的一批 TIFF。若保存的裁剪范围超出当前 TIFF，程序会提示错误并拒绝加载，不会静默改变裁剪区域。预设只保存参数，不复制 TIFF 数据或批处理输出文件。

## TIFF 文件夹批量处理

V0.5.0 起可以一次处理一个文件夹顶层的全部 `.tif` 和 `.tiff` 文件：

1. 点击“打开 TIFF 文件夹”，程序会按文件名排序并列出所有 TIFF。
2. 选择一个 TIFF 作为参考文件，点击“载入选中的参考 TIFF”。
3. 在参考文件上设置裁剪范围、相颜色/透明度/平滑、边框、3D 文字、相机角度、截图倍率和逐层体积分数公式。
4. 点击“批量输出 PNG 和体积分数 CSV”，选择结果保存位置。

程序会把参考 TIFF 的同一套裁剪、显示、相机、边框和 3D 文字设置应用到每个文件。所有文件都必须在去除单例维度后具有相同的原始 `(Z, Y, X)` 形状；形状不一致或读取失败的文件会跳过并记录原因，不会中断其他文件。

每次运行都会新建一个不与旧结果冲突的 `phase_viewer_batch_output` 目录（若已存在则自动添加编号），其中包含：

~~~text
phase_viewer_batch_output/
├── visualizations/                       每个 TIFF 一张同视角 PNG
├── slice_volume_fraction/                每个 TIFF 一份逐层 CSV
├── all_slice_volume_fraction_X|Y|Z.csv   全部成功文件的合并结果
├── batch_manifest.csv                    成功/失败状态及对应输出文件
└── batch_settings.csv                    参考文件、裁剪和计算参数
~~~

CSV 中的 `Original Slice Number` 是原始体数据的 1-based 层号；即使参考文件从中间开始裁剪，编号也不会从 1 重新开始。批量计算始终使用裁剪后的全分辨率矩阵，不受三维预览降采样影响。

## 裁剪工作区域

“裁剪工作区域”使用原始体数据的 X/Y/Z 体素索引，每个方向都可以设置起始和结束值：

- 索引从 1 开始
- 起始和结束值都包含在裁剪范围中
- 点击“应用裁剪”后，三维表面、边框、截图和逐层体积分数都会改用裁剪后的工作体积
- 点击“恢复完整范围”可回到原始矩阵

例如原始矩阵的 X×Y×Z 为 `200×200×200`，要显示 `180×200×200`，可设置：

~~~text
X: 1 到 180
Y: 1 到 200
Z: 1 到 200
~~~

也可以修改起始值来截取任意偏移区域。裁剪不会重新编号空间坐标，因此逐层计算结果仍可以追溯到原始体数据。

## 多相显示

相列表中每一行对应一个相：

- “显示”复选框：即时显示或隐藏该相
- “颜色”按钮：打开颜色选择器
- “透明度”：0 表示完全透明，1 表示完全不透明
- “平滑”复选框：控制是否平滑该相
- “光照”复选框：独立启用或关闭该相的表面光照

首次载入时，程序将推测体素数最多的标签为背景相并隐藏它，默认显示最多前 8 个非背景相。可以使用“非背景相”“全部显示”和“全部隐藏”批量控制。

## 每个相的独立属性

在左侧单击一行，然后在右侧“当前相属性”中设置：

- 表面颜色
- 透明度
- 是否启用平滑
- 平滑迭代次数
- 松弛因子
- 最大位移
- Lighting 开关
- Flat、Gouraud 或 Phong 表面插值
- Ambient、Diffuse、Specular 和 Specular Power

点击“应用并重建当前相”只会重新计算当前相，不会重建其他相。

由于各相可以使用不同的平滑强度，极端参数可能使相邻相表面产生少量交叉或间隙。科学分析时建议相邻相使用相同参数，并限制最大位移。

## 逐层体积分数

右侧独立的“计算 / Slice Volume Fraction”面板始终可见，默认计算：

~~~text
count(phase == 3) / count(phase in [3, 4, 5])
~~~

可以选择 X、Y 或 Z 方向，也可以修改分子相编号和分母相编号列表。计算使用当前裁剪工作区域的全分辨率矩阵，不受三维预览降采样影响。

结果会在底部表格中显示两列：

- Original Slice Number：该层在原始体数据中的 1-based 切片编号
- Volume Fraction：该层的计算比例

如果某层完全没有分母相，其结果显示为 N/A。表格可以保存为 CSV。

## 模型边框

当前工作区域外侧会显示一个三维线框。右侧面板可以：

- 显示或隐藏边框
- 使用颜色选择器修改边框颜色
- 在 0.5 到 12.0 之间修改边框粗细

边框始终对应当前裁剪范围，不包含 Surface Nets 使用的内部填充层。

## 3D 文字区域标注

“3D 文字区域标注”支持同时管理多条标注：

- 使用自由的 X/Y/Z 模型世界坐标指定位置，三个方向都允许负数
- 一键使用当前裁剪区域中心
- 独立设置文字颜色和字号
- 可选择 Times New Roman、Arial 或 Courier New 英文字体
- 每条标注可以独立启用加粗和斜体
- 新增、更新、删除或清空标注
- 文字使用 VTK Billboard 方式渲染，会始终朝向相机
- 在列表中选择标注后，可以拖动文字锚点处的彩色球形手柄；X/Y/Z 数值会实时更新
- 按住 Shift 拖动手柄时，可沿最接近拖动方向的坐标轴约束移动

标注不再受裁剪范围限制，即使位于模型框外也会保持显示。可以取消“显示所选标注的拖动手柄”来隐藏手柄；保存模型截图时程序也会自动临时隐藏手柄，因此截图只包含模型、边框和 3D 文字。

## 背景与模型截图

右侧面板可以：

- 设置纯色或渐变背景
- 设置工作区域边框颜色和粗细
- 显示三角网格边
- 重置相机
- 选择 1×、2× 或 4× 截图清晰度
- 保存仅包含中央模型渲染区域的 PNG

导出 2× 或 4× 高分辨率截图时，程序会按相同倍率临时放大 3D 标注字体，因此导出图片中文字与模型的相对大小会和交互界面保持一致。导出完成后，界面中的字号仍保持用户设置值，例如 22 号。

截图不包含相列表、Properties 面板或体积分数表格。

## TIFF 数据要求

- 去除单例维度后必须是三维矩阵
- 程序按 (Z, Y, X) 理解数组，第一个维度通常对应 TIFF 页
- 数值必须是离散整数相编号，例如 0、1、2
- 连续灰度 TIFF 需要先完成分相或分割
- 原型最多接受 512 个不同标签

## 当前限制

- 完整 TIFF 会一次读入内存；部分裁剪还会建立一个连续的工作数组
- 每个可见相会分别生成表面；相很多时应先隐藏不需要的相
- 当前裁剪通过数值索引控制，尚未加入可拖动的 Box Widget 手柄
- 未加入正交切片、任意方向裁剪平面、整体体积/表面积统计和网格导出
- TIFF 中的物理 spacing 当前主要由用户明确输入
- 文件夹批处理只扫描所选文件夹顶层，不递归读取子文件夹
